# mk-on-guide

一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面的存储与展示空间。

## 项目简介

`mk-on-guide` 旨在收集、整理并托管一系列独立的 HTML 页面。这些页面可能是教程、工具、演示或参考文档，每个页面均可独立访问，无需依赖后端服务。

## 目录说明

```
mk-on-guide/
├── index.html          # 仓库首页/导航页
├── pages/              # 存放所有独立 HTML 页面
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
└── assets/             # 公共资源文件（CSS、JS、图片等）
    ├── styles/
    ├── scripts/
    └── images/
```

- `pages/`：每个 HTML 文件为一个独立页面，文件名建议使用英文小写及连字符。
- `assets/`：存放页面共用的样式、脚本及图片资源，避免重复。

## 页面归档说明

- 所有页面均为纯前端 HTML，不包含服务器端逻辑。
- 页面之间无强关联，可单独通过 URL 访问。
- 若页面需引用外部资源（如字体库、图标库），建议使用 CDN 或本地化存储。

## 维护说明

- 新增页面：在 `pages/` 目录下创建 HTML 文件，并在 `index.html` 中添加链接（可选）。
- 更新页面：直接修改对应文件，注意保持资源引用路径正确。
- 删除页面：移除文件及 `index.html` 中对应链接。
- 提交前请确保 HTML 语法正确，资源路径有效。
